# OaNnAutogen

Schema-driven generator for `OaModule` layer wrappers.

This generator owns mechanical NN layer boilerplate. It intentionally does not
own `OaModule`, optimizer behavior, runtime dispatch, or graph policy.

Generated layer files are split one layer per file:

```text
Source/Public/Oa/Ml/Nn/Relu/Relu.h
Source/Private/Oa/Ml/Nn/Relu/Relu.cpp
```

The generated header is the public module declaration; it is not wrapped by
an otherwise-empty hand-written header. `Source/Public/Oa/Ml/Nn/Layers.h`
aggregates those declarations for the public `Oa/Ml/Nn.h` umbrella. Generated
implementations and `NnSources.gen.cmake` remain private.

Public semantic files use normal API names regardless of whether they are
generated. The ownership banner, schema source, idempotent writer, stale-output
cleanup, and drift gate enforce generated ownership; `.gen` is reserved for
internal mechanical artifacts such as the CMake source manifest.

Layer names use PascalCase without acronym spelling:

```text
Relu, Gelu, Silu, Softmax, RmsNorm
```

Run:

```bash
python3 Tools/NnAutogen/oannautogen.py --live
```

Live generation always consumes the complete schema set because the shared
`Layers.h` and `NnSources.gen.cmake` manifests must describe every layer.
`--schema` is therefore preview-only and is rejected with `--live`. Unchanged
generated bytes preserve their existing timestamps, so an idempotent run does
not invalidate the build graph. A complete run also removes stale files carrying
this generator's ownership marker.

The VS Code task `Ml - OaFnAutoGen` runs both `OaFnAutogen` and `OaNnAutogen`.
