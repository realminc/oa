# OaTypeAutogen

Schema-driven code generator for OA C++ types (enums, structs). Stdlib Python only — no pip deps, runs anywhere with Python 3.11+.

## Status

**Type generator** — generates enums and structs from TOML schemas under `Tools/TypeAutogen/Schema/`.

Each schema emits:
  - `.gen.h` declarations for enums and structs
  - `.gen.cpp` implementations (ToString, FromString, serialization, validation)

## Usage

```bash
# Generate all schemas to Tools/TypeAutogen/Output/
python3 Tools/TypeAutogen/oatypeautogen.py

# Dry run — print what would be written
python3 Tools/TypeAutogen/oatypeautogen.py --dry-run

# Write generated files into Source/
python3 Tools/TypeAutogen/oatypeautogen.py --live

# Generate specific schema
python3 Tools/TypeAutogen/oatypeautogen.py \
    --schema Tools/TypeAutogen/Schema/Core/CoreTypes.toml
```

## Layout

  - `Schema/<Domain>/*.toml` — source of truth, one file per domain
  - `oatypeautogen.py` — CLI entrypoint and code emitters
  - `oatypeautogen_lib/` — config, schema validation, emitters
  - `Output/` — generated files, gitignored, drop-in replacements for hand-written files in `Source/`
  - `README.md` — this file

## File Structure

Generated files follow the OaFnMatrix pattern:
  - `Source/Public/Oa/<Domain>/Type.gen.h` — generated public declarations
  - `Source/Private/Oa/<Domain>/Type.gen.cpp` — generated implementation
  - `Source/Public/Oa/<Domain>/Type.h` — public non-generated header that includes Type.gen.h

This keeps declarations on the public include path and generated implementations in
Private/. Do not add a second `Type.gen.h` under Private/: `Source/Private` is an
include root for the `oa` library, so a duplicate would shadow the schema-generated
public header for library sources.

## Schema Format

```toml
# Tools/TypeAutogen/Schema/Core/CoreTypes.toml
domain = "Core"
namespace = "Oa"

[[enums]]
name = "OaScalarType"
underlying_type = "OaU8"
generate_tostring = true
generate_fromstring = true
values = [
    { name = "Float32", value = 0, comment = "32-bit float" },
    { name = "BFloat16", value = 1, comment = "Brain float 16" },
    { name = "Float16", value = 2, comment = "Half precision" },
    # ...
]

[[structs]]
name = "OaViewerConfig"
generate_serialize = true
generate_validate = true
fields = [
    { name = "Width", type = "OaU32", default = "1920" },
    { name = "Height", type = "OaU32", default = "1080" },
    { name = "Mode", type = "OaViewerMode", default = "OaViewerMode::Default" },
]
```

## Generated Output

**Public generated header (`Source/Public/Oa/Core/Type.gen.h`):**
```cpp
// AUTO-GENERATED by Tools/TypeAutogen/oatypeautogen.py — DO NOT EDIT.
#pragma once
#include <Oa/Core/Types.h>

enum class OaScalarType : OaU8 {
    Float32 = 0,  // 32-bit float
    BFloat16 = 1,  // Brain float 16
    // ...
};
[[nodiscard]] constexpr const char* OaScalarTypeToString(OaScalarType InType) noexcept;
[[nodiscard]] OaScalarType OaScalarTypeFromString(const char* InStr) noexcept;
```

**Public header (`Source/Public/Oa/Core/Type.h`):**
```cpp
// OA Core Types
#pragma once
#include <Oa/Core/Type.gen.h>
```

**Implementation (`Source/Private/Oa/Core/Type.gen.cpp`):**
```cpp
// AUTO-GENERATED by Tools/TypeAutogen/oatypeautogen.py — DO NOT EDIT.
#include <Oa/Core/Type.gen.h>
#include <cstring>

constexpr const char* OaScalarTypeToString(OaScalarType InType) noexcept {
    switch (InType) {
        case OaScalarType::Float32: return "Float32";
        case OaScalarType::BFloat16: return "BFloat16";
        // ...
        default: return "unknown";
    }
}

OaScalarType OaScalarTypeFromString(const char* InStr) noexcept {
    if (strcmp(InStr, "Float32") == 0) return OaScalarType::Float32;
    if (strcmp(InStr, "BFloat16") == 0) return OaScalarType::BFloat16;
    // ...
    return OaScalarType::Float32;  // Default to first value
}
```

## Adding a New Type

1. Add `[[enums]]` or `[[structs]]` entry to the appropriate domain's TOML schema
2. Run the generator: `python3 Tools/TypeAutogen/oatypeautogen.py --live`
3. (Optional) Update CMake to include the generated `.gen.cpp` file

## Domain Split

Types are organized by domain to enable modular builds:
  - **Core**: Foundational types (OaScalarType, OaPrecision, OaDeterminismMode)
  - **Ml**: ML-specific types (OaActivation, OaPrecisionTag, OamCompression)
  - **Vision**: Image/video types (OaPixelFormat, OaVideoCodec, OaImageFormat)
  - **Ui**: UI types (OaViewerMode, OuiKey, OuiEventType)
  - **Runtime**: GPU/runtime types (OaGemmKernel, OaStoragePrecision)
  - **Audio**: schema-owned Audio enums and POD configuration types
  - **Render**: Render types (OaLightType, OaMeshTopology, OaBlendMode)
  - **Animation**: Animation types (OaTimeDirection, OaInterpolationMode)

This enables building subsets like:
  - Core + Runtime (minimal compute)
  - Core + Ml (ML only)
  - Core + Vision (vision only)
  - All (full stack)

## Unified Generator

Use `Tools/oaautogen.py` to run all generators at once:

```bash
# Generate all (types + functions)
python3 Tools/oaautogen.py

# Generate types only
python3 Tools/oaautogen.py --types

# Generate functions only
python3 Tools/oaautogen.py --functions

# Write to Source/
python3 Tools/oaautogen.py --live
```
